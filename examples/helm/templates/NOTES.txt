## Monitoring the Deployment

Check the status of your operator:

```bash
# Check ClusterExtension status
kubectl get clusterextension {{ .Values.operator.name | default .Release.Name }} -n {{ .Values.operator.namespace }}

# Check operator pods
kubectl get pods -n {{ .Values.operator.namespace }}

# Check operator logs
kubectl logs -n {{ .Values.operator.namespace }} -l app.kubernetes.io/name={{ .Values.operator.name | default .Release.Name }}
```

## Troubleshooting

If you encounter issues:

1. **Check ClusterExtension status**:
   ```bash
   kubectl describe clusterextension {{ .Values.operator.name | default .Release.Name }} -n {{ .Values.operator.namespace }}
   ```

2. **Check operator logs**:
   ```bash
   kubectl logs -n {{ .Values.operator.namespace }} -l app.kubernetes.io/name={{ .Values.operator.name | default .Release.Name }}
   ```

3. **Verify RBAC permissions**:
   ```bash
   kubectl get clusterrolebinding | grep {{ .Release.Name }}
   kubectl get rolebinding -n {{ .Values.operator.namespace }} | grep {{ .Release.Name }}
   ```

## Uninstalling

To uninstall the release:

```bash
helm uninstall {{ .Release.Name }} -n {{ .Values.operator.namespace }}
```

**Note**: This will remove the ClusterExtension, which will trigger OLM to uninstall the operator and clean up associated resources.

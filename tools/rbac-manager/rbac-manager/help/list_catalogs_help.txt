List ClusterCatalogs Command

USAGE:
    rbac-manager list-catalogs [FLAGS]

DESCRIPTION:
    Lists all available ClusterCatalogs in the OpenShift cluster.
    Shows catalog status, source type, and image references.

REQUIRED FLAGS:
    --skip-tls            Required for clusters with self-signed certificates

OPTIONAL FLAGS:
    --debug               Enable debug logging

EXAMPLES:
    # List catalogs (requires --skip-tls for self-signed certs)
    rbac-manager list-catalogs --skip-tls

    # List catalogs with debug output
    rbac-manager list-catalogs --skip-tls --debug

OUTPUT:
    The command displays a table showing:
    - Catalog Name
    - Status (Ready/Unknown/Error)
    - Source Image Reference
    - Source Type

NOTES:
    - This command requires a valid kubeconfig or OpenShift credentials
    - For clusters with self-signed certificates, --skip-tls is mandatory
    - The command queries the olm.operatorframework.io/v1/ClusterCatalogs API
    - Catalogs must be in Ready status to be used with other commands

TROUBLESHOOTING:
    - "No ClusterCatalogs found": Ensure OLMv1 is installed and catalogs are created
    - "Failed to list ClusterCatalogs": Check kubeconfig and cluster connectivity
    - "SSL certificate verify failed": Use --skip-tls flag for self-signed certificates
